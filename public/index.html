Whenime API

User Routes

Example request body: {
"username": "johndoe",
"email": "johndoe@example.com",
"password": "securepassword123",
"date_of_birth": "2000-01-15",
"profile_picture": "johndoe.jpg",
"about": "I'm a software engineering student passionate about web development and UX design.",
"is_admin": 0
}

GET /users - Get all users

Query parameters: search <string> | limit <int> | offset <int>

Example: /users?search=me&limit=2&offset=1

Response types: id <int> | username <string> | date_of_birth <date> | profile_picture <string> | about <string> | email <string> | email_verified_at <date> | password <string> | is_admin <int> | remember_token <string> | created_at <date> | updated_at <date>

GET /users/{id} - Get user by ID

Parameters: id <int>

Response types: id <int> | username <string> | date_of_birth <date> | profile_picture <string> | about <string> | email <string> | email_verified_at <date> | password <string> | is_admin <int> | remember_token <string> | created_at <date> | updated_at <date>

POST /users - Add a user

Request body: username <string> | email <string> | password <string> | date_of_birth <date> (optional) | profile_picture <string> (optional) | about <string> (optional) | is_admin <int> (optional)

PATCH /users/{id} - Update user by ID

Parameters: id <int>

Request body (all optional): username <string> | date_of_birth <date> | profile_picture <string> | about <string> | email <string> | password <string> | is_admin <int>

DELETE /users/{id} - Delete user by ID

Parameters: id <int>

News Routes

Example request body: {
"title": "New Features Released in Our App",
"image": "app-update.jpg",
"content": "We are excited to announce new features in our app, including improved UI, enhanced performance, and more customization options."
}

GET /news - Get all news

Response types: id <int> | title <string> | image <string> | content <string> | created_at <date> | updated_at <date>

GET /news/{id} - Get news by ID

Parameters: id <int>

Response types: id <int> | title <string> | image <string> | content <string> | created_at <date> | updated_at <date>

POST /news - Add news item

Request Body: title <string> | image <string> | content <string>

PATCH /news/{id} - Update news item by ID

Parameters: id <int>

Request body (all optional): title <string> | image <string> | content <string>

DELETE /news/{id} - Delete news item by ID

Parameters: id <int>

Contact Form Routes

Example request body: {
"name": "John Doe",
"email": "johndoe@example.com",
"subject": "Issue with App Login",
"message": "I'm unable to log into the app despite entering the correct credentials. Please assist.",
"status": "UNREAD"
}

GET /forms - Get all contact forms

Response types: id <int> | name <string> | email <string> | subject <string> | message <string> | status <enum('UNREAD', 'READ', 'SOLVED')> | created_at <date> | updated_at <date>

GET /forms/{id} - Get form by ID

Parameters: id <int>

Response types: id <int> | name <string> | email <string> | subject <string> | message <string> | status <enum('UNREAD', 'READ', 'SOLVED')> | created_at <date> | updated_at <date>

POST /forms - Add contact form submission

Request body: name <string> | email <string> | subject <string> | message <string>

PATCH /forms/{id} - Update form by ID

Parameters: id <int>

Request body (all optional): name <string> | email <string> | subject <string> | message <string> | status <enum('UNREAD', 'READ', 'SOLVED')>

DELETE /forms/{id} - Delete form by ID

Parameters: id <int>
